Adicionando o Agente como serviço.
a. Instale o agente conforme descrito acima.

b. Criar um link para /etc/sysconfig
• [appdynamics]# ln -s <machine-agent-home> /etc/sysconfig/appdynamics-machine-agent /etc/sysconfig/appdynamics-machine-agent

c. Adicionar o Machine Agent para /etc/init.d, conforme exemplo abaixo.
• [appdynamics]# ln -s <machine-agent-home> /etc/init.d/appdynamics-machine-agent /etc/init.d/appdynamics-machine-agent

d. Editar a variável no arquivo de configuração /etc/sysconfig/appdynamics-machine-agent
• MACHINE_AGENT_HOME: Especifica onde o arquivo Machine Agent esta localizado. Exemplo: /opt/AppDynamics
• JAVA_HOME: Especifica o JRE utilizado. Exemplo: /opt/AppDynamics/machineagent/jre
• MACHINE_AGENT_USER: Especifique o usuário do sistema no qual é usado para iniciar o Machine Agent. Por padrão ele é root. AppDynamics recomenda que você crie um usuário non-root para rodar o Machine Agent. O novo usuário deve ter “read-access” para o arquivo controller-info.xml e escrita para acesar o arquivo de log. Exemplo: MACHINE_AGENT_USER=root

e. Adicione o agente como serviço. Por exemplo, por linha de comando.
• [appdynamics]# chkconfig --add appdynamics-machine-agent

f. Inicie o serviço do agente.
• [appdynamics]# service appdynamics-machine-agent start ou /etc/init.d/appdynamics-machineagent start

g. Pedir para a equipe de monitoração verificar se o agente esta reportando para a Controller.
